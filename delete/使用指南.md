# 韩新乐的博客 - 使用指南

## 📋 项目概述

这是一个基于 Jekyll + Forty 主题的个人学术博客，支持技术文章与摄影内容展示。

---

## 🚀 本地运行

### 前置要求

1. **安装 Ruby**（Jekyll 依赖）
   - Windows: 下载 [RubyInstaller](https://rubyinstaller.org/)（推荐 Ruby+Devkit 3.2.x）
   - 安装时勾选 "Add Ruby executables to your PATH"
   - 安装完成后运行 `ridk install`，选择 MSYS2 and MINGW development toolchain

2. **安装 Jekyll 和 Bundler**
   ```bash
   gem install jekyll bundler
   ```

### 启动本地服务器

```bash
# 1. 安装依赖（首次运行）
bundle install

# 2. 启动本地服务器
bundle exec jekyll serve

# 3. 浏览器访问
# http://localhost:4000
```

## ✍️ 如何添加新文章

### 技术博客

1. 在 `_posts/` 目录创建文件，命名格式：`YYYY-MM-DD-标题.md`

   例如：`2026-03-22-python-tutorial.md`

2. 文件开头添加 Front Matter：

```yaml
---
layout: post
title: "Python 教程"
date: 2026-03-22 10:00:00 +0800
categories: [tech]
tags: [python, tutorial]
image: assets/images/pic02.jpg
description: 这是一篇 Python 教程
---
```

3. 正文使用 Markdown 格式编写

### 摄影文章

1. 同样在 `_posts/` 创建文件

2. Front Matter 示例：

```yaml
---
layout: post
title: "街头摄影作品"
date: 2026-03-22 12:00:00 +0800
categories: [photography]
tags: [street, black-and-white]
image: assets/images/your-photo.jpg
description: 街头摄影作品集
---
```

3. 图片放在 `assets/images/` 目录
4. 文章中引用图片：`![描述]({{ site.baseurl }}/assets/images/your-photo.jpg)`

## 📤 部署到 GitHub Pages

### 方法 1：GitHub Actions（推荐）

1. 在仓库 Settings → Pages 中：
   - Source: GitHub Actions
   - 选择 Jekyll 工作流

2. 推送代码后自动构建部署：
```bash
git add .
git commit -m "Update site"
git push origin master
```

### 方法 2：手动构建

```bash
# 1. 本地构建
bundle exec jekyll build

# 2. 推送 _site 目录到 gh-pages 分支
# （需要额外配置）
```

## 📝 内容迁移

你的 29 篇 Hexo 文章已备份在：
```
_backup_hexo_20260322_233349/source/_posts/
```

迁移步骤：
1. 打开备份目录中的 `.md` 文件
2. 调整 Front Matter 格式（Hexo → Jekyll）
3. 重命名为 `YYYY-MM-DD-标题.md`
4. 移动到 `_posts/` 目录

**Hexo vs Jekyll Front Matter 对比**：

```yaml
# Hexo 格式
---
title: 文章标题
date: 2022-12-14 18:27:04
tags: [tag1, tag2]
---

# Jekyll 格式（需要改成这样）
---
layout: post
title: "文章标题"
date: 2022-12-14 18:27:04 +0800
categories: [tech]
tags: [tag1, tag2]
image: assets/images/pic01.jpg
---
```

## 🆘 常见问题

### 1. Ruby 未安装
- 下载 [RubyInstaller](https://rubyinstaller.org/)
- 安装后运行 `ridk install`

### 2. bundle install 失败
```bash
# 更换国内镜像源
bundle config mirror.https://rubygems.org https://gems.ruby-china.com
bundle install
```

### 3. 图片不显示
- 检查图片路径是否正确
- 使用 `{{ site.baseurl }}/assets/images/xxx.jpg`

### 4. 本地预览正常但部署后样式错误
- 检查 `_config.yml` 中的 `baseurl` 和 `url` 配置
